# デプロイ後のチェックリスト

## ✅ デプロイ完了後の確認事項

### 1. サイトが正常に表示されるか確認
1. Renderから発行されたURL（例: `https://suiren.onrender.com`）にアクセス
2. ログイン画面が表示されることを確認
3. エラーが出ていないか確認

### 2. 新規登録ができるか確認
1. 「新規登録」をクリック
2. ユーザー名とパスワードを入力して登録
3. 登録が成功し、自動的にログインされることを確認

### 3. ログインができるか確認
1. 一度ログアウト
2. 登録したユーザー名とパスワードでログイン
3. ログインが成功することを確認

### 4. 支払い機能をテスト
1. ログイン後、支払い金額を入力（例: 1000円）
2. 「PayPayで支払う」をクリック
3. モーダルが表示されることを確認
4. 「支払い完了」をクリック
5. ポイントカードに金額が追加されることを確認

### 5. QRコード生成機能をテスト
1. 「売り手用 - QRコード生成」セクションで金額を入力
2. 「QRコードを生成」をクリック
3. QRコードが表示されることを確認
4. 「画像をダウンロード」をクリックして、ダウンロードできることを確認

### 6. データが保存されるか確認
1. ブラウザを閉じて再度開く
2. 同じユーザーでログイン
3. ポイントカードのデータが残っていることを確認

## 🚀 実際に使うための準備

### 1. URLを共有
- Renderから発行されたURLを学校内で共有
- 例: `https://suiren.onrender.com`

### 2. 使い方を説明
以下の使い方をユーザーに説明：

**初回利用者:**
1. サイトにアクセス
2. 「新規登録」でユーザー名とパスワードを設定
3. 登録後、自動的にログインされます

**買い手:**
1. ログイン
2. 支払い金額を入力
3. 「PayPayで支払う」をクリック
4. PayPayで実際に支払い
5. 「支払い完了」をクリックして記録

**売り手:**
1. ログイン
2. 「売り手用 - QRコード生成」で金額を入力
3. QRコードを生成して印刷またはダウンロード
4. 買い手にQRコードを見せて、PayPayでスキャンしてもらう

## ⚠️ 注意事項

### Renderの無料プランの制限
- **スリープ機能**: 15分間アクセスがないと自動的にスリープします
- **起動時間**: スリープ後、次回アクセス時に約30秒かかります
- **月間時間**: 750時間/月（約31日間）
- **帯域幅**: 100GB/月

### データの保存について
- データはRenderのサーバーに保存されます
- 無料プランでもデータは保持されます
- より永続的な保存が必要な場合は、RenderのPostgreSQL（有料）を検討してください

### セキュリティ
- パスワードはハッシュ化されて保存されます
- HTTPSで通信が暗号化されます
- ただし、実際のPayPay決済は外部で行われるため、このシステムは記録のみを行います

## 🔧 トラブルシューティング

### サイトが表示されない
- Renderのダッシュボードでサービスの状態を確認
- ログを確認してエラーがないか確認
- ビルドが成功しているか確認

### ログインできない
- ユーザー名とパスワードが正しいか確認
- データベースが正常に作成されているか確認
- サーバーのログを確認

### データが保存されない
- ブラウザのコンソールでエラーを確認
- サーバーのログを確認
- データベースファイルが作成されているか確認

### スリープからの起動が遅い
- 無料プランの制限です
- 定期的にアクセスがあるとスリープしません
- 有料プランにアップグレードすると常時起動できます
