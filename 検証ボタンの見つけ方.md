# 検証ボタンの見つけ方

## 🔍 検証ボタンはどこにある？

検証ボタンは、LINE Developers Consoleの「Messaging API」ページにあります。

## 📋 検証ボタンを見つける手順

### ステップ1: LINE Developers Consoleにアクセス

1. ブラウザで [LINE Developers Console](https://developers.line.biz/console/) を開く
2. LINEアカウントでログイン

### ステップ2: プロバイダーとチャネルを選択

1. あなたのプロバイダー（例：すいれん）をクリック
2. チャネルID `2008552055` のチャネルをクリック

### ステップ3: Messaging APIページを開く

1. **左側のメニュー**を確認
2. **「Messaging API」**をクリック
   - 「チャネル基本設定」の下にあります
   - アイコンは💬のような形です

### ステップ4: Webhook URLセクションを探す

1. ページを開いたら、**下にスクロール**してください
2. **「Webhook URL」**というセクションを探します
   - ページの上の方ではなく、**下の方**にあります
   - スクロールしないと見えない場合があります

### ステップ5: 検証ボタンを探す

「Webhook URL」セクションの中に、以下の要素があります：

1. **入力欄**：Webhook URLを入力する場所
2. **「更新」ボタン**：URLを保存するボタン
3. **「検証」ボタン**：URLを検証するボタン

「検証」ボタンは、「更新」ボタンの**右側**または**下側**にあります。

## 🔍 見つからない場合の確認ポイント

### ポイント1: 正しいページにいるか確認

- 左メニューで「Messaging API」をクリックしているか
- 「チャネル基本設定」ではなく、「Messaging API」ページにいるか

### ポイント2: ページを下にスクロール

- 「Webhook URL」セクションは、ページの**下の方**にあります
- 上から下まで、しっかりスクロールしてください

### ポイント3: ブラウザの検索機能を使う

1. ブラウザで `Ctrl + F`（Windows）または `Cmd + F`（Mac）を押す
2. 「検証」と入力して検索
3. 検証ボタンがハイライトされます

### ポイント4: 別のタブやセクションを確認

- 「Messaging API」ページには複数のセクションがあります
- 「Webhook URL」セクションを探してください
- 他のセクション（例：「応答メッセージ」）ではありません

## 📋 具体的な手順（もう一度）

### 1. LINE Developers Consoleを開く

```
https://developers.line.biz/console/
```

### 2. プロバイダーを選択

- あなたのプロバイダー（例：すいれん）をクリック

### 3. チャネルを選択

- チャネルID `2008552055` のチャネルをクリック

### 4. 左メニューから「Messaging API」をクリック

- 左側のメニューに「Messaging API」があります
- クリックしてください

### 5. ページを下にスクロール

- マウスホイールを下に回す
- または、ページの右側のスクロールバーを下に動かす

### 6. 「Webhook URL」セクションを探す

- 「Webhook URL」という見出しを探す
- その下に、入力欄とボタンがあります

### 7. 「検証」ボタンを探す

- 「更新」ボタンの近くに「検証」ボタンがあります
- 右側または下側にあります

## 💡 それでも見つからない場合

### 確認事項

1. **正しいチャネルを選択しているか**
   - チャネルID `2008552055` のチャネルを選択しているか

2. **「Messaging API」ページにいるか**
   - 左メニューで「Messaging API」をクリックしているか
   - URLに `/messaging-api` が含まれているか

3. **ページが完全に読み込まれているか**
   - ページをリロード（F5キー）してみる

4. **別のブラウザで試す**
   - Chrome、Firefox、Safariなど、別のブラウザで試す
