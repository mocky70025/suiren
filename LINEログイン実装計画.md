# LINEログイン実装計画

## 🎯 目標

LINE公式アカウントを使うため、LINEユーザーIDでアカウントを管理できるようにする。

## 📋 実装内容

### 1. LINEログイン（OAuth）の実装
- LINE Login APIを使用
- ユーザーがLINEアカウントでログインできるようにする

### 2. LINEユーザーIDでアカウント管理
- LINEユーザーIDを主キーとして使用
- 既存のユーザー名/パスワードログインも残す（互換性のため）

### 3. データベースの拡張
- `line_user_id`カラムは既に追加済み
- LINEユーザーIDでユーザーを検索・作成する機能を追加

### 4. フロントエンド統合
- LINEログインボタンを追加
- LINEログイン後の処理を実装

## 🔧 実装手順

### ステップ1: LINE Login設定
1. LINE Developers ConsoleでLINE Loginを有効化
2. コールバックURLを設定
3. チャネルIDとチャネルシークレットを取得

### ステップ2: サーバー側の実装
1. LINE Login OAuthエンドポイントを実装
2. コールバック処理を実装
3. LINEユーザーIDでユーザーを作成・認証

### ステップ3: フロントエンド統合
1. LINEログインボタンを追加
2. ログイン後の処理を実装

## 💡 アーキテクチャ

### 現在の方式
```
ユーザー名 + パスワード → ログイン
```

### 新しい方式
```
LINEアカウント → LINEログイン → LINEユーザーIDで管理
```

### 両方対応
```
ユーザー名 + パスワード → ログイン（既存）
LINEアカウント → LINEログイン → LINEユーザーIDで管理（新規）
```
